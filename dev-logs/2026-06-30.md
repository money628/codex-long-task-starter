# 开发日志 - 2026-06-30

## 今日完成
- 优化 AI 访谈定位：明确报告读者是 Codex/OpenCode 这类可以操作电脑的 Agent，而不是让普通用户回答技术方案。
- 访谈 prompt 增加规则：扫码登录、cookie/session 保存、抓取策略、部署方式等技术细节默认交给 Agent 推荐，不再反复追问代码小白。
- 访谈 prompt 增加问题去重和最多 3 个问题的约束。
- AI 包增加问题去重保险：过滤历史中已问过的问题，并限制每轮最多 3 个问题。
- ProjectSpec 生成 prompt 增加 Agent 读者说明，把用户不懂的技术细节放到 recommendations / unresolvedQuestions。
- 访谈页新增提示：报告会交给 Codex/OpenCode 执行，用户只需确认目标、边界和验收标准。
- 访谈问题新增“交给 Agent 推荐”按钮，避免用户被迫回答专业实现细节。
- 未提交的访谈输入会自动保存到 localStorage，误点切页后返回不会丢。
- 点击“继续访谈”进入下一轮后会自动滚动回问题顶部。
- 增强重复问题识别：按主题过滤“权重/点赞/收藏/评论/关注/私信”等同类复盘问题，而不只比较完全相同文本。
- 当模型返回结构不合法且修复失败时，访谈不再卡死，改为本地兜底收口问题并允许生成初版 ProjectSpec。
- “交给 Agent 推荐”按钮改为未选中时中性样式，只有用户点击后才高亮。
- 新增 AI 访谈回归测试：同主题重复问题过滤、结构校验失败兜底。
- 用真实 Edge Profile + DeepSeek 配置跑通 Web 全流程：创建项目、AI 访谈、ProjectSpec、Markdown 结果页。
- 真实 Web 流程发现 DeepSeek 仍会问“使用哪种 AI 模型”“是否登录注册”等代码小白不该决策的问题。
- 增加代码层过滤：代码小白 + 本地工具场景下过滤模型选择、登录注册、数据库、部署、cookie/session 等实现问题。
- 新增回归测试：代码小白场景会过滤模型选择和登录注册等实现问题。
