# 设计规范

## 视觉原则
- 保留 Stitch 设计稿的深色 Codex 工作台风格。
- 不重写 UI，不推翻现有布局、卡片、侧栏、顶部导航和结果页结构。
- 新增能力优先接入现有组件，小范围适配输入框、状态和按钮。

## 中文化规则
- 面向用户的按钮、导航、状态、提示和面板标题使用中文。
- 技术标识可以保留英文：API Key、Base URL、ProjectSpec、Markdown、CLI、JSON、文件名、供应商名称、命令。
- 中文文案要短，避免撑破按钮和卡片。

## 交互规则
- 无 API Key 时明确展示“示例模式”。
- 真实 API 模式必须让用户知道本工具不提供模型额度。
- 下载、复制、导出等动作保持在结果页工具栏中，不新增复杂入口。

## 禁止事项
- 不使用全新设计系统替换现有页面。
- 不添加登录、付费、数据库等与当前阶段无关的页面。
- 不把密钥、Token 或账号信息展示在日志、Markdown 或导出文件中。
