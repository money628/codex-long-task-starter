# 产品定位与商业化判断

本文档用于判断 Codex 长任务启动器 / Codex Long Task Starter 在开源 alpha 阶段之后，应该继续打磨还是进入商业化验证。

## 一句话定位

Codex 长任务启动器（Codex Long Task Starter）是一个 local-first 的 AI 编程长任务启动器，帮助用户把模糊项目想法整理成 Codex / OpenCode 可以执行的任务文件包。

## 当前阶段判断

当前更适合做“开源 alpha + 轻商业验证”，不建议立刻转成完整 SaaS。

原因：

- 核心闭环已经跑通：配置模型、AI 访谈、ProjectSpec、Markdown、ZIP、CLI。
- 真实用户试用样本还少，访谈质量、提示词深度、不同 Provider 稳定性仍需要观察。
- 目标用户是否愿意付费，付费点是模板、咨询、工作流包还是托管服务，目前还需要验证。
- local-first 是当前差异化优势，过早 SaaS 化会增加登录、数据库、计费、隐私和运维负担。

## 适合的第一批用户

- 想用 Codex / OpenCode 做完整项目，但不知道如何写任务启动文档的人。
- 有产品想法但代码经验不足，希望 AI 帮忙整理需求的人。
- 独立开发者、小团队、运营人员、内容创业者。
- 经常使用 AI 编程工具，但容易遇到上下文断裂和任务跑偏的人。

## 当前核心价值

- 把“我有一个想法”变成结构化 ProjectSpec。
- 把 ProjectSpec 变成 7 个 Agent 可读 Markdown 文件。
- 本地使用自己的 API Key，降低隐私和平台依赖。
- 用 CLI 把任务包写入目标项目，方便交给 Codex / OpenCode 继续执行。

## 轻商业验证方向

优先考虑低成本验证：

1. 开源版本免费
   - 保持 GitHub 仓库可运行。
   - 收集真实反馈和 star。

2. 付费模板包
   - 例如小红书工具、Chrome 插件、独立站、AI 工作流、内部管理系统。
   - 每个模板包含更好的访谈问题、ProjectSpec 示例和执行文件。

3. 付费陪跑或定制
   - 帮用户把一个真实项目整理成 Codex / OpenCode 任务包。
   - 观察他们真正卡在哪一步。

4. 后续再考虑 Pro 版本
   - 云端保存项目。
   - 多设备同步。
   - 团队协作。
   - 更稳定的模型代理。

## 暂不做

- 用户登录。
- 云数据库。
- SaaS 计费系统。
- 团队协作。
- 模板市场。
- Electron 桌面端。
- Docker 化。

## 下一阶段成功指标

- 至少 10 个真实用户跑通 Web 全流程。
- 至少 5 个用户把导出的 Markdown 交给 Codex / OpenCode 执行。
- 至少 3 个用户愿意提交反馈或 issue。
- 至少 1 个用户愿意为模板包、定制任务包或陪跑付费。
- 不同 Provider 下，核心流程没有明显阻塞。

## 建议路线

1. 先发布 `v0.1.0-alpha.1` GitHub Release。
2. 用中文内容做第一轮推广，收集试用反馈。
3. 根据反馈优先修复访谈重复、问题太专业、导出文件不够可执行的问题。
4. 做 2-3 个更贴近普通用户的示例模板。
5. 如果有人愿意付费，再设计最小商业版本，而不是先做完整 SaaS。
